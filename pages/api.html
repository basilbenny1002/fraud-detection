Overview

The FraudShield API provides endpoints for detecting fraud in websites and financial transactions. All API requests require authentication via API key and return JSON responses.

Authentication

All API requests require an API key to be included in the request headers.

Headers 
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

Website Fraud Check

Analyze a website for potential fraud indicators based on various parameters.

POST /website-check

Request Parameters

Parameter Type Required Description
website_url string Yes The URL of the website to analyze
credit_card_payment boolean Yes Presence of credit card payment option
money_back_payment boolean Yes Presence of money back guarantee
cash_on_delivery boolean Yes Cash on delivery payment option
crypto_currency boolean Yes Cryptocurrency payment option
free_contact_emails boolean Yes Use of free email services for contact
logo_url boolean Yes Presence of company logo

Example Request

cURL 
curl -X POST http://localhost:8000/website-check \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "website_url": "https://example-shop.com",
    "credit_card_payment": true,
    "money_back_payment": false,
    "cash_on_delivery": true,
    "crypto_currency": false,
    "free_contact_emails": true,
    "logo_url": false
  }'

Example Response

JSON 
{
  "status_code": 200,
  "prediction": 1,
  "confidence": 0.87,
  "result": "fraudulent",
  "message": "Website shows high fraud indicators",
  "timestamp": "2025-01-06T12:55:08Z"
}

Transaction Fraud Check

Analyze financial transactions for fraud detection using machine learning models.

POST /transaction-check

Request Parameters

Parameter Type Required Description
amount number Yes Transaction amount
oldbalanceOrg number Yes Original account balance before transaction
newbalanceOrig number Yes New account balance after transaction
oldbalanceDest number Yes Destination account balance before transaction
newbalanceDest number Yes Destination account balance after transaction
isFlaggedFraud number Yes System flag (0 or 1)
type string Yes Transaction type: CASH_IN, CASH_OUT, DEBIT, PAYMENT, TRANSFER

Example Request

cURL 
curl -X POST http://localhost:8000/transaction-check \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000.00,
    "oldbalanceOrg": 15000.00,
    "newbalanceOrig": 10000.00,
    "oldbalanceDest": 2000.00,
    "newbalanceDest": 7000.00,
    "isFlaggedFraud": 0,
    "type": "TRANSFER"
  }'

Example Response

JSON 
{
  "status_code": 200,
  "prediction": 0,
  "confidence": 0.92,
  "result": "legitimate",
  "message": "Transaction appears legitimate",
  "timestamp": "2025-01-06T12:55:08Z"
}

Response Format

All API responses follow a consistent JSON format.

Field Type Description
status_code number HTTP status code (200 for success)
prediction number 0 = legitimate, 1 = fraudulent
confidence number Confidence score (0.0 to 1.0)
result string Human-readable result: "legitimate" or "fraudulent"
message string Descriptive message about the result
timestamp string ISO 8601 timestamp of the analysis

Error Codes

The API uses standard HTTP status codes to indicate success or failure.

Status Code Description Example Response
200 Success Request processed successfully
400 Bad Request Invalid parameters or missing required fields
401 Unauthorized Invalid or missing API key
429 Rate Limited Too many requests, please slow down
500 Server Error Internal server error, please try again

Error Response Format

JSON 
{
  "status_code": 400,
  "error": "Bad Request",
  "message": "Missing required parameter: amount",
  "timestamp": "2025-01-06T12:55:08Z"
}

Rate Limits

API requests are rate limited to ensure fair usage and system stability.

Current Limits

  • Free Tier: 100 requests per hour
  • Pro Tier: 1,000 requests per hour
  • Enterprise: Custom limits

Rate limit information is included in response headers:

Headers 
X-RateLimit-Limit: 100
